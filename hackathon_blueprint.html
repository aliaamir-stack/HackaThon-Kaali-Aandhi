01

Problem & Solution

⚠ The Problem

Patients in rural areas lack access to specialized healthcare. Community health workers have no formal diagnostic training. Language barriers and low literacy rates make it impossible to accurately convey symptoms or translate them into structured clinical data.

✓ The Solution

A voice-first multi-agent diagnostic pipeline. Health workers record native-language voice descriptions. A team of 6 AI agents translates, structures, cross-references, and safety-checks the data before generating a formatted referral note for a human doctor. Never an autonomous diagnosis.

Each Agent — Detailed Breakdown

🎙 Transcription Tool
TRIGGER / TOOL — Not an LLM agent
Model
Whisper Large-v3 Turbo (via API) or whisper.cpp locally
Input
Audio file (WAV/MP3) in regional language
Output
Raw text string (may be Urdu/Sindhi/Pashto etc.)
Role
Pure audio→text. No reasoning. Just transcription.
Key tip
Use Groq Whisper API — fastest inference, free tier.
🌐 Localization Agent
AGENT 1 — Custom addition, high impact
Model
Llama-3-8B-Instruct or Mixtral 8x7B via Groq
Input
Raw transcribed text in regional language
Output
JSON: { "translated_text": "...", "source_language": "..." }
Role
Translate to clinical English. At end of pipeline, translate final note BACK to native language for patient.
Key tip
Prompt: "You are a medical translator. Preserve all symptom details precisely."
📋 Triage Agent
AGENT 2 — Core medical agent
Model
DeepSeek-R1 or GPT-4o via API
Input
Clinical English text from Localization Agent
Output
Structured JSON: symptoms[], duration, severity, vitals_if_known, missing_info[]
Role
Structure the complaint. Identify what information is missing. Do NOT diagnose.
Key tip
Handle ambiguity gracefully — if symptom is unclear, add to missing_info[] rather than guess.
🔬 Specialist Agent
AGENT 3 — RAG-powered reasoning
Model
DeepSeek-R1 + FAISS/ChromaDB (WHO/CDC guidelines)
Input
Structured JSON from Triage Agent
Output
JSON: potential_conditions[], recommended_tests[], urgency_level (1-5), evidence_sources[]
Role
Cross-reference symptoms against medical knowledge base. Flag potential conditions with evidence. Never diagnose.
Key tip
RAG retrieval is the key differentiator here — prevents hallucination and impresses judges.
🚨 Safety Agent
AGENT 4 — Parallel observer, system override
Model
DeepSeek-R1 or Claude API
Input
Outputs from BOTH Triage AND Specialist agents
Output
JSON: is_urgent (bool), red_flags[], drug_interactions[], override_required (bool)
Role
Detect: chest pain, stroke symptoms, sepsis, dangerous combos. If override_required=true → skip to Summary with URGENT flag.
Key tip
This is the most impressive agent for judges — shows safety-first design thinking.
📄 Summary Agent
AGENT 5 — Final output generator
Model
DeepSeek-R1 or GPT-4o
Input
Specialist output + Safety output (or URGENT flag from Safety override)
Output
Formatted referral note (Markdown) in English + native language translation
Role
Synthesize everything into a clear doctor-ready referral note. Call Localization Agent to back-translate for patient.
Key tip
Include: Patient data, symptoms, potential flags, urgency level, recommended next steps.

The State JSON — Shared Pipeline Object

This single object is initialized by the Orchestrator and passed through every agent. Each agent reads from it and writes its output back into it. Pydantic enforces the schema at every step.

# state.py — Pydantic Model class PipelineState(BaseModel): # Input layer audio_path: str raw_transcript: str = "" source_language: str = "" clinical_english: str = "" # Triage layer symptoms: list[str] = [] duration: str = "" severity: int = 0 # 1-10 missing_info: list[str] = [] # Specialist layer potential_conditions: list[str] = [] urgency_level: int = 1 # 1-5 evidence_sources: list[str] = [] # Safety layer is_urgent: bool = False red_flags: list[str] = [] override_required: bool = False # Output layer referral_note_en: str = "" referral_note_native: str = "" pipeline_status: str = "pending"
# graph.py — LangGraph State Machine from langgraph.graph import StateGraph graph = StateGraph(PipelineState) # Add all nodes graph.add_node("transcribe", transcribe_node) graph.add_node("localize", localize_node) graph.add_node("triage", triage_node) graph.add_node("specialist", specialist_node) graph.add_node("safety", safety_node) graph.add_node("summary", summary_node) # Linear flow graph.add_edge("transcribe", "localize") graph.add_edge("localize", "triage") graph.add_edge("triage", "specialist") graph.add_edge("specialist", "safety") # Safety conditional override graph.add_conditional_edges( "safety", route_after_safety, # checks override_required {"urgent": "summary", "normal": "summary"} ) graph.set_entry_point("transcribe") graph.set_finish_point("summary")

671B reasoning model for Triage, Specialist, Safety, Summary agents. Access via SiliconFlow or Together AI. Free/cheap inference.

Medical Knowledge Base
FAISS + ChromaDB
WHO Primary Healthcare Guidelines + CDC Rural Health PDFs converted to vector store. Specialist Agent queries this to prevent hallucinations.

Pydantic v2
Strict JSON output validation at every agent handoff. Prevents pipeline crashes from malformed LLM responses.
🔑
Config & Secrets
python-dotenv
All API keys in .env file. Never hardcode. Load with os.getenv(). Add .env to .gitignore immediately.

Critical Rules to Follow

01
Never let an agent diagnose. Every medical agent must include in its system prompt: "You do not issue diagnoses. You flag potential conditions for a human doctor to evaluate." This is the human-in-the-loop requirement and a judging criterion.
02
Pydantic on every output. Wrap every LLM response in a try/except and validate against the Pydantic model. If the model returns malformed JSON, retry once with a "correct your JSON format" prompt before failing gracefully.
03
Build agents one at a time. Don't wire the full graph until each individual agent works standalone. Test each agent with a hardcoded input before connecting it to the previous agent's output.
04
RAG before prompt engineering. Get the FAISS index built in Hours 6–8. A Specialist Agent with RAG grounding is 10x more impressive than one with a long medical system prompt. It's also safer.
05
Log everything. Every agent should print/log its input and output during execution. This serves two purposes: debugging during build, and making the agent pipeline visually explainable in the demo.
06
Prepare two test cases. A normal case (fever, cough, fatigue) and an urgent case (chest pain, difficulty breathing, sweating). Show both in the demo to demonstrate the Safety override triggering correctly.
